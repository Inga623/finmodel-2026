# Финансовая модель 2026

Помесячная P&L-модель по маркетплейсам **WB** и **Ozon**: заказы, выкуп, сезонность, выручка, переменные расходы, маржа, ФОТ, УСН, НДС/НБ, денежный поток, сценарии (база / оптимистичный / стресс).

## Запуск

```bash
npm install
npm start
```

Приложение будет доступно на порту **3000** (или на порту из переменной окружения `PORT`).

## Структура

- **`public/calc.js`** — ядро модели: одна функция `calculateModel(params)`, все расчёты только здесь.
- **`public/`** — фронт: форма параметров, таблица, графики, экспорт CSV/Excel.
- **`server/index.js`** — Express: раздача статики и API (сценарии, расчёт).

## API

### POST /api/calculate

Принимает JSON с параметрами модели, возвращает результат расчёта (выручка, переменные, маржа, оpex, чистая прибыль, кеш и т.д.).

Пример минимального тела запроса (остальное подставится по умолчанию):

```json
{
  "wbPrice": 2310,
  "wbOrders": 21100,
  "wbBuyoutRate": 100,
  "ozPrice": 3200,
  "ozOrders": 25000,
  "ozBuyoutRate": 100,
  "scenario": "base"
}
```

Опционально можно передать `scenarioOverrides`: `{ "sales": 1.1, "cost": -0.5, "ads": -1 }` для кастомных сдвигов сценария.

Ответ — объект с полями: `months`, `revenue`, `variable`, `margin`, `opex`, `net`, `taxBase`, `taxBaseCumul`, `vat7`, `vat22`, `cash`, `bepMonth`, `bepRevenue` и др.

### Сценарии

- **GET /api/scenarios** — список сохранённых сценариев.
- **POST /api/scenarios** — сохранить сценарий (`name`, `params`).
- **GET /api/scenarios/:id** — загрузить сценарий по id.

## Окружение

- `PORT` — порт сервера (по умолчанию 3000).
- `NODE_ENV=production` — отключает логирование запросов к `/api/calculate`.

## Версионирование модели

Поддерживается версия **v1** (параметр `version` в запросе или по умолчанию). Входные данные проверяются перед расчётом; при ошибке API возвращает 400 с полем `error`.
